# Implementation Plan 02: DB

**Parent:** [full-project.plan.md](full-project.plan.md) — §3.6 packages/db, §5 Database & Multi-Org Structure, §5.1 Database Architecture

**Related:** [03 Validators](03-validators.plan.md) (Zod schemas use types aligned with DB; include `version` in card update); [10 Analytics](10-analytics.plan.md) (analytics_events, analytics_daily_summary); [roadmap.md](roadmap.md) (v3 full version history)

## Scope

`packages/db` — Drizzle schema, migrations, Studio. All core tables with indexes and constraints.

## Deliverables

- `packages/db/package.json`, `tsconfig.json`
- `packages/db/src/schema/` — users, organizations, organization_members, cards, short_links, review_cards, qr_records, analytics_events, org_entitlements, media, tags, templates, organization_invites, activity_log
- Indexes per structure plan (slug, short_code, org_id, created_at, etc.)
- `cards.version` — integer, incremented on every update; used for optimistic concurrency (sync and API return 409 if client sends stale version). Full version history table (e.g. card_versions with JSON snapshot) deferred to v3; see [roadmap.md](roadmap.md).
- Soft deletes: `deleted_at` on cards, short_links, qr_records, review_cards, media
- `packages/db/src/migrate.ts`
- `packages/db/drizzle.config.ts`
- `turbo.json` — db:generate, db:migrate, db:studio tasks

## Key Tables


| Table                | Key columns                                                |
| -------------------- | ---------------------------------------------------------- |
| users                | id, email, ... (Better Auth)                               |
| organizations        | id, slug, name                                             |
| organization_members | userId, organizationId, role                               |
| cards                | id, organizationId, slug, data, template, theme, version (integer, for optimistic concurrency / sync conflict) |
| short_links          | id, organizationId, shortCode, targetUrl, expiresAt        |
| review_cards         | id, organizationId, slug, reviewUrl, template, theme       |
| qr_records           | id, organizationId, type, payload                          |
| analytics_events     | id, organizationId, entityType, entityId, event, timestamp |
| org_entitlements     | organizationId, productId, polarOrderId                    |
